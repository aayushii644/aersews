# Concrete Compressive Strength Predictor

A **research-grade machine-learning pipeline** built around the classic concrete dataset.  
The project trains, compares and tunes a variety of regression models to estimate the **compressive strength (MPa)** of concrete mixtures, then exposes the best-performing model via a lightweight Flask/AJAX UI.

> Includes baseline models, hyperparameter search, test-set isolation, plots, and a web UI  
> Written in pure Python with scikit-learn, XGBoost, LightGBM, CatBoost & Flask

1. **Dataset**

## Running the training & evaluation pipeline

The core script is \main.py\; it:

1. loads & sanitises the data (1,030 samples, 8 features, 9 columns total)
2. splits into a held-out test set (824 train / 206 test)
3. evaluates 14 baseline models (linear, tree-based, kernels, boosting, etc.)
4. performs randomized hyperparameter tuning on selected learners
5. builds comparison tables and saves CSVs
6. serialises the **best model**, and
7. produces two plots (\prediction_vs_actual.png\ and \eature_importance.png\)

\\\powershell
python main.py
\\\

All results are written under \outputs/\ - metrics, plots and \est_model.pkl\.

> The script is already configured to use a fixed random seed (\42\) for reproducibility.  
> There is no quick-run toggle anymore; the current hyperparameter search sizes are moderate and finish in a few minutes on a modern desktop.

## Evaluation metrics & models

* **Metrics**: MAE, RMSE, R² and adjusted R²
* **Baseline learners (14 models)**: Linear, Ridge, Lasso, ElasticNet, DecisionTree, RandomForest, ExtraTrees, GradientBoosting, XGBoost, LightGBM, CatBoost, SVR, KNN, MLP
* **Tuned learners**: CatBoost and RandomForest (extended RandomizedSearchCV)

After tuning the best models are compared alongside the top-5 baselines in \inal_leaderboard.csv\.

## Web application

A simple single-page frontend lets users enter mixture parameters and get a strength prediction.

1. Ensure \outputs/models/best_model.pkl\ exists (run \main.py\ first).
2. Launch Flask:

   \\\ash
   python app.py
   \\\

3. Open [http://127.0.0.1:5000](http://127.0.0.1:5000) in your browser.

The AJAX form sends eight numeric features (cement, slag, fly ash, water, superplasticizer, coarse aggregate, fine aggregate, age) to \/predict\ and displays the rounded MPa estimate.

## Reproducibility & improvements

The codebase includes a changelog comment block at the top of \main.py\ listing:

* Correct log-uniform parameterisation for CatBoost learning_rate
* Removal of early_stopping_rounds from CV (breaks RandomizedSearchCV)
* Consistent column naming across DataFrames
* Explicit test-set isolation guard (never touched in model selection)
* Fixed regex warnings in pandas replacements
* Added MAE to all results tables

These improvements were made after initial experiments to ensure reliable, research-grade output.
